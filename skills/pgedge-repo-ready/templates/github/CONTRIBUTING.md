# Contributing to <PROJECT_NAME>

Thank you for your interest in contributing to <PROJECT_NAME>.

## Getting Started

1. Fork the repository
2. Create a feature branch (`feat/your-feature` or
   `fix/your-fix`)
3. Make your changes
4. Run tests (`make test`)
5. Run linting (`make lint`)
6. Submit a pull request

## Development Setup

See the [README](README.md) for prerequisites and setup
instructions.

## Pull Request Guidelines

- One logical change per PR
- Follow conventional commit style for your PR title
  (`feat:`, `fix:`, `docs:`, `chore:`, etc.)
- Include tests for new functionality
- Ensure all CI checks pass before requesting review
- Update documentation if behavior changes

## Code Style

- See `CLAUDE.md` for project-specific conventions
- Run `make lint` before submitting

## Reporting Issues

- Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md)
  for bugs
- Use the [feature request template](.github/ISSUE_TEMPLATE/feature_request.md)
  for new features
